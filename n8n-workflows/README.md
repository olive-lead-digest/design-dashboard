## The workflows

| File | Workflow | Trigger (verified from live workflows) | Purpose | Supabase tables touched |
|---|---|---|---|---|
| `1-feasibility-generator.json` | DD-1 Feasibility Study Generator | Webhook `POST /webhook/feasibility-generator` | Fetches the project, generates an AI (Claude) feasibility study, saves it, and logs a communication row | `projects` (read), `feasibility_studies` (insert), `communications` (insert), `audit_logs` (insert on failure) |
| `2-floor-plan-generator.json` | DD-2 Floor Plan Generator | Webhook `POST /webhook/floor-plan-generator` | Generates a deterministic 2-band floor plan (2D SVG data URI + 3D box-model JSON data URI + dimensions/materials breakdowns), versions it per project, saves it, logs a communication row | `projects` (read), `floor_plans` (read + insert), `communications` (insert) |
| `3-communication-logger.json` | DD-3 Communication Logger | Webhook `POST /webhook/communication-logger` | Normalizes an update event and writes a communications row (`sent_to_owner = false`) plus an audit trail entry (`communication_logged`) | `communications` (insert), `audit_logs` (insert) |
| `4-task-accountability.json` | DD-4 Task Accountability Tracker | Schedule: daily 09:00 (set workflow timezone to Asia/Kolkata — see Manual steps) | Computes per-member task metrics (totals, completion rate, overdue count, avg completion time), marks past-due tasks as `Overdue`, writes ONE daily audit row (`accountability_daily_check`) | `team_tasks` (read + update), `audit_logs` (insert) |
| `5-projections-generator.json` | DD-5 Projections Generator | Schedule: weekly Monday 09:00 (set workflow timezone to Asia/Kolkata) **AND** Webhook `POST /webhook/projections-generator` (on-demand; responds immediately with `{"status":"accepted"}`) | Aggregates the portfolio (projects + investments + tasks), asks Claude for 4 strict-JSON projections (Timeline, Budget, ROI, Risk), inserts them, writes an audit row (`projections_generated` / `projections_generation_failed`) | `projects`, `investment_requirements`, `team_tasks` (read), `projections` (insert), `audit_logs` (insert) |

Webhook URLs (once active): `https://olivehospitality.app.n8n.cloud/webhook/<path>` (production) or `.../webhook-test/<path>` (test mode).

Note on the schedule triggers: the live DD-4 and DD-5 schedules are configured for 09:00 daily / Monday 09:00 respectively, but the workflow **timezone is not set programmatically** — set it to `Asia/Kolkata` in each workflow's Settings (otherwise they fire at 09:00 in the instance default timezone).

3. **Importing into another n8n instance.** In the n8n editor: **Workflow menu (⋯) → Import from File** and pick one of these JSON files (repeat per file). After import, open each Supabase node and select/re-create the Supabase credential, and open the Anthropic node in DD-1/DD-5 and attach the Anthropic credential (credential IDs differ per instance, so they must be reattached). Save, set timezone where relevant, then activate.
